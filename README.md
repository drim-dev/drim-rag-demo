# RAG Practice Applications

Три практических приложения для изучения Retrieval-Augmented Generation (RAG). Каждое приложение — самостоятельный проект с нарастающей сложностью.

## Предварительные требования

- [uv](https://docs.astral.sh/uv/) — менеджер пакетов Python
- Python 3.11+
- Docker Desktop (для приложений 2 и 3)
- Git

## Приложения

### App 1: Ask Paul Graham

**Уровень:** Начальный · **Время на запуск:** ~3 мин · **Стоимость:** ~$0.05

Простое RAG-приложение для поиска по эссе Пола Грэма. Вы освоите базовый RAG-пайплайн: загрузка документов, чанкинг, эмбеддинги, векторный поиск и генерация ответа.

**Стек:** LlamaIndex · ChromaDB · OpenAI · Streamlit

**Что вы изучите:**
- Чанкинг документов с настраиваемым размером
- Генерация эмбеддингов и векторное хранилище
- Поиск по косинусному сходству (similarity search)
- Аугментация промпта контекстом

→ [`app-1-ask-paul-graham/`](app-1-ask-paul-graham/)

---

### App 2: Python Docs Assistant

**Уровень:** Средний · **Время на запуск:** ~5 мин · **Стоимость:** ~$0.30

Ассистент по документации Python со сравнением стратегий поиска. Вы увидите, почему наивный RAG ломается на структурированных документах, и научитесь это исправлять.

**Стек:** LangChain · Qdrant · Cohere + OpenAI · Gradio

**Что вы изучите:**
- Гибридный поиск: векторный + BM25 с Reciprocal Rank Fusion
- Реранкинг результатов (Cohere rerank)
- HyDE — трансформация запроса через гипотетический ответ
- Фильтрация по метаданным (модуль, тип документации)
- Parent-child retrieval

→ [`app-2-python-docs-assistant/`](app-2-python-docs-assistant/)

---

### App 3: K8s Knowledge Base

**Уровень:** Продвинутый · **Время на запуск:** ~10 мин · **Стоимость:** ~$0.50

Продакшн-ready RAG-система с тремя индексами (документация, исходный код, API-спеки), автоматической маршрутизацией запросов, оценкой качества (RAGAS) и мониторингом.

**Стек:** LlamaIndex · pgvector · Ollama + Claude · Streamlit

**Что вы изучите:**
- Мультииндексная архитектура с маршрутизацией запросов
- Code-aware чанкинг (tree-sitter для Go)
- Оценка качества через RAGAS (faithfulness, relevancy, precision)
- Семантическое кэширование
- Трекинг стоимости и задержек

→ [`app-3-k8s-knowledge-base/`](app-3-k8s-knowledge-base/)

---

## Рекомендуемый порядок

1. **App 1** — получите первый "RAG работает!" момент за 50 строк кода
2. **App 2** — поймите, почему наивный подход ломается, и как это чинить
3. **App 3** — постройте систему продакшн-уровня с оценкой и мониторингом

Каждое приложение независимо — можно начать с любого, но порядок 1 → 2 → 3 даёт лучший learning experience.
