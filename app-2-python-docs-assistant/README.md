# Python Docs Assistant — RAG-ассистент по документации Python

## Что вы построите

RAG-приложение, которое отвечает на вопросы по официальной документации Python. В отличие от простого RAG с единственным vector search, здесь вы реализуете продвинутый retrieval-пайплайн с несколькими методами поиска и сможете сравнить их эффективность.

Приложение индексирует RST-файлы из репозитория CPython (stdlib reference + tutorial) и предоставляет Gradio-интерфейс для поиска и генерации ответов.

## Чему вы научитесь

- **Hybrid search (RRF)** — объединение vector search и BM25 через Reciprocal Rank Fusion
- **HyDE** — Hypothetical Document Embeddings: генерация гипотетического ответа для улучшения поиска
- **Reranking** — переранжирование результатов с помощью Cohere Rerank
- **Header-based chunking** — разбиение RST-документов по секциям с сохранением иерархии
- **Semantic chunking** — группировка текста по смысловой близости
- **Parent-child retrieval** — поиск по дочерним чанкам, возврат родительских для полного контекста
- **Metadata filtering** — фильтрация по модулю и типу документа

## Предварительные требования

- [uv](https://docs.astral.sh/uv/) — менеджер пакетов Python
- Python 3.11+
- Docker (для Qdrant)
- API-ключи: [OpenAI](https://platform.openai.com/api-keys) и [Cohere](https://dashboard.cohere.com/api-keys)

## Быстрый старт

1. **Установите зависимости:**
   ```bash
   uv sync
   ```

2. **Настройте API-ключи:**
   ```bash
   cp .env.example .env
   # Отредактируйте .env — добавьте ваши ключи OpenAI и Cohere
   ```

3. **Запустите Qdrant:**
   ```bash
   docker compose up -d
   ```

4. **Скачайте документацию Python:**
   ```bash
   uv run python download_data.py
   ```

5. **Постройте индекс:**
   ```bash
   uv run python index.py --strategy header-based
   ```

   Доступные стратегии: `fixed`, `header-based`, `semantic`

6. **Запустите приложение:**
   ```bash
   uv run python app.py
   ```

   Откройте http://localhost:7860 в браузере.

### Пайплайн индексации

1. **Загрузка** — скачивание RST-файлов из репозитория CPython (stdlib + tutorial)
2. **Парсинг** — извлечение секций с заголовками, сохранение иерархии (модуль > класс > метод)
3. **Чанкинг** — разбиение на части одной из трёх стратегий
4. **Метаданные** — имя модуля, путь секции, тип документа
5. **Эмбеддинги** — Cohere embed-v4 через LangChain
6. **Хранение** — Qdrant (векторы) + BM25 индекс (pickle)

### Методы поиска

| Метод | Описание |
|-------|----------|
| **Vector search** | Косинусное сходство эмбеддингов запроса и чанков |
| **BM25** | Классический keyword-matching, хорош для точных имён API |
| **Hybrid (RRF)** | Объединение vector + BM25 через Reciprocal Rank Fusion |
| **HyDE** | LLM генерирует гипотетический ответ → его эмбеддинг используется для поиска |
| **Reranking** | Cohere rerank переоценивает top-K результатов |
| **Parent-child** | Поиск по мелким чанкам, возврат родительской секции для контекста |

## Упражнения

1. **Сравните стратегии чанкинга.** Переиндексируйте с `--strategy fixed`, затем с `--strategy header-based`. Задайте запрос "Как работает система импортов Python?" и сравните качество ответов. Обратите внимание, как header-based чанкинг сохраняет логическую структуру документации.

2. **BM25 vs Vector для точных имён API.** Попробуйте запрос "asyncio.Lock" с vector-only и hybrid поиском. BM25 лучше находит точные совпадения имён API, а vector search может "промахнуться", найдя семантически похожие, но не те разделы.

3. **Эффект реранкинга.** Включите реранкинг и сравните порядок результатов до и после. Обратите внимание, как Cohere Rerank переставляет чанки, оценивая релевантность пары (запрос, документ) более точно, чем простое косинусное сходство.
