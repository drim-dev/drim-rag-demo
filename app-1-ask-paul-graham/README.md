# Ask Paul Graham — RAG Practice App

## Что вы построите

Приложение для вопросов и ответов по эссе Пола Грэма. Вы загружаете эссе, разбиваете их на фрагменты, строите векторный индекс и задаёте вопросы на естественном языке. Модель отвечает только на основе найденных фрагментов и указывает источники.

## Чему вы научитесь

- **Chunking** — как разбить длинные тексты на фрагменты и почему размер фрагмента влияет на качество ответов
- **Embedding** — как тексты превращаются в векторы и зачем это нужно для поиска по смыслу
- **Similarity search** — как находить релевантные фрагменты по вопросу пользователя
- **Prompt augmentation** — как подставить найденный контекст в промпт, чтобы модель отвечала по фактам, а не галлюцинировала

## Предварительные требования

- [uv](https://docs.astral.sh/uv/) — менеджер пакетов Python
- [Docker](https://www.docker.com/) — для запуска Qdrant
- Python 3.11+
- Ключ OpenAI API (нужен доступ к `text-embedding-3-small` и `gpt-4o-mini`)

## Быстрый старт

1. Установите зависимости:

   ```bash
   uv sync
   ```

2. Скопируйте `.env.example` в `.env` и вставьте свой ключ:

   ```bash
   cp .env.example .env
   # Отредактируйте .env — укажите OPENAI_API_KEY
   ```

3. Запустите Qdrant:

   ```bash
   docker compose up -d
   ```

   Дашборд Qdrant доступен по адресу: http://localhost:6333/dashboard

4. Скачайте эссе:

   ```bash
   uv run python download_data.py
   ```

5. Запустите приложение:

   ```bash
   uv run streamlit run app.py
   ```

6. В боковой панели нажмите **"Построить индекс"**, затем задавайте вопросы.

## Как это работает

1. **Загрузка данных** — скрипт `download_data.py` скачивает эссе Пола Грэма с paulgraham.com и сохраняет как текстовые файлы.

2. **Chunking** — тексты разбиваются на фрагменты заданного размера (по умолчанию 512 токенов) с перекрытием. Перекрытие нужно, чтобы не терять контекст на границах фрагментов.

3. **Embedding** — каждый фрагмент преобразуется в вектор размерности 1536 с помощью модели `text-embedding-3-small`. Векторы сохраняются в Qdrant.

4. **Поиск** — при вопросе пользователя вопрос тоже превращается в вектор. Qdrant находит top-K ближайших фрагментов по косинусному расстоянию.

5. **Генерация ответа** — найденные фрагменты подставляются в промпт, и `gpt-4o-mini` генерирует ответ только на основе предоставленного контекста.

## Упражнения

### 1. Эксперимент с chunk size

Измените chunk size с 512 на 128 через селектор в боковой панели и перестройте индекс. Задайте тот же вопрос и сравните ответы. Обратите внимание: при маленьких фрагментах контекст обрезается и модели не хватает информации для полного ответа.

### 2. Top-K: precision vs recall

Установите top-K в 1 и задайте широкий вопрос (например, "What does Paul Graham think about startups?"). Затем поставьте top-K в 10 и повторите. С малым K ответ точнее, но может упустить важные аспекты. С большим K ответ полнее, но может включить нерелевантный контекст.

### 3. Вопрос за пределами данных

Задайте вопрос о событиях после 2023 года (например, "What does Paul Graham think about GPT-5?"). Наблюдайте: хорошо настроенный RAG скажет, что не имеет информации. Если модель начинает "выдумывать" — это галлюцинация, и промпт нужно доработать.
